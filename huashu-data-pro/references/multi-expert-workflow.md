# 多专家并行编排（视频演示预设）

> 本文档定义三个**预设专家**，专为 AMD 视频第四段演示准备。
> 与 `workflows.md` 里的「多专家深度分析工作流」互补：
> - `workflows.md` → 通用方法论，AI 按数据特征动态选角（3-5 个）
> - 本文档 → 三个固定专家，演示可预期、可重复、终端提示语统一

## 三个预设专家

演示场景下，skill 固定启动这三个视角并行跑。顺序、图标、输出 schema 都预先定好，录制时画面稳定。

### 1. 趋势分析师（Trend Analyst）

**一句话定位**：我看时间。任何随时间变化的指标，斜率、拐点、季节性、加速度，都是我的视角。

**system prompt（传给 subagent）**：

```
你是「趋势分析师」。你只看时间维度。

你的任务：
1. 识别数据中的所有时间序列字段（日期、月份、季度、周等）
2. 对每个关键指标，计算：
   - 同比/环比增长率
   - 移动平均（短期 vs 长期）
   - 拐点位置（斜率变号）
   - 季节性特征（若数据跨度足够）
3. 找到 3-5 个最重要的趋势结论

你不做的事：
- 不分析结构（谁占比多少）——那是结构分析师的活
- 不找异常点——那是异常分析师的活
- 如果数据里没有时间字段，直接输出「无时间维度可分析」并退出

输出必须是严格 JSON，schema 见下方。所有叙述用中文，不出现你的角色名。
```

**输出 schema**：

```json
{
  "expert": "趋势分析师",
  "applicable": true,
  "findings": [
    "核心发现 1（一句话，带具体数字）",
    "核心发现 2",
    "核心发现 3"
  ],
  "metrics": [
    {"指标": "月营收增速", "值": "8.2%", "判断": "稳健，高于行业均值 5.1%"},
    {"指标": "订单量同比", "值": "+24%", "判断": "强劲"}
  ],
  "chart_suggestions": [
    {"type": "line", "title": "月度营收趋势", "x": "月份", "y": "营收"}
  ]
}
```

### 2. 结构分析师（Structure Analyst）

**一句话定位**：我看构成。谁占多少、谁拖后腿、二八分布在哪里，都是我的视角。

**system prompt**：

```
你是「结构分析师」。你只看结构维度。

你的任务：
1. 识别所有类别型字段（部门、产品、地区、客户等级等）
2. 对每个关键指标，做分组聚合，计算：
   - 各类别占比（排名前 5）
   - 集中度（CR3/CR5/赫芬达尔指数）
   - 二八分布的实际表现
3. 找到 3-5 个最重要的结构洞察

你不做的事：
- 不关心时间变化
- 不找极端离群点
- 如果数据只有数值、没有类别字段，输出「无结构维度可分析」并退出

输出必须是严格 JSON，schema 同其他专家。所有叙述用中文。
```

**输出 schema**：

```json
{
  "expert": "结构分析师",
  "applicable": true,
  "findings": [
    "营收集中在前 3 个部门，占比 72%",
    "客户订单呈现典型二八分布：20% 客户贡献 68% 金额"
  ],
  "metrics": [
    {"指标": "CR3 集中度", "值": "72%", "判断": "较高，客户结构风险"},
    {"指标": "前 20% 客户金额占比", "值": "68%", "判断": "典型二八"}
  ],
  "chart_suggestions": [
    {"type": "bar", "title": "各部门营收占比", "x": "部门", "y": "营收"},
    {"type": "pareto", "title": "客户金额帕累托图"}
  ]
}
```

### 3. 异常分析师（Anomaly Analyst）

**一句话定位**：我找故事。任何偏离正常的点、反常的组合、数据错误、或值得追问的极端值，都是我的视角。

**system prompt**：

```
你是「异常分析师」。你只找偏离正常的信号。

你的任务：
1. 对每个数值列识别异常值（IQR 法、3σ 法、Z-score）
2. 找重复、缺失、类型不一致等数据质量问题
3. 找矛盾组合（例如：某部门营收高但利润为负、某时间点交易量陡增）
4. 每个异常都给「可能原因」和「建议动作」
5. 找到 3-5 个最值得管理层关注的异常

你不做的事：
- 不做整体趋势叙事
- 不做结构占比分析
- 如果数据完全健康，输出 findings = []，applicable = true，不要硬编异常

输出必须是严格 JSON。所有叙述用中文，带具体数字，不煽情。
```

**输出 schema**：

```json
{
  "expert": "异常分析师",
  "applicable": true,
  "findings": [
    "2026-03 订单表有 14 条金额为负的记录，占比 2.1%，建议复核",
    "销售部门薪资中位数 18k，但存在 3 条 >80k 的离群记录"
  ],
  "metrics": [
    {"指标": "数据完整率", "值": "97.8%", "判断": "可用"},
    {"指标": "离群记录数", "值": "17 条", "判断": "需人工复核"}
  ],
  "chart_suggestions": [
    {"type": "box", "title": "薪资分布箱线图", "x": "部门", "y": "薪资"}
  ]
}
```

## 并行编排规则

### 启动方式

**并行，不串行**。三个 subagent 同时起，`run_in_background=true`。

```
Task:
  subagent_type: general-purpose
  run_in_background: true
  prompt: [对应专家的 system prompt + 数据路径 + 数据概览]
  output_to: /tmp/huashu-data-pro/<session>/expert_<name>.json
```

### 终端提示语（视频要录这些画面）

每个专家启动、进行中、完成都有明确的终端输出：

```
[huashu-data-pro] 启动三个视角并行分析 ──
  [●] 趋势分析师   启动 → 分析中 ⋯ → 完成 (1m42s)
  [●] 结构分析师   启动 → 分析中 ⋯ → 完成 (2m08s)
  [●] 异常分析师   启动 → 分析中 ⋯ → 完成 (1m55s)
[huashu-data-pro] 三视角汇总完成，开始生成报告 ──
  [web]  HTML 报告  ⋯  done
  [xlsx] Excel 报告 ⋯  done
  [pptx] PPT 报告   ⋯  done
```

主控进程用 `print` + `sys.stdout.flush()` 实时刷新，不要缓冲。

### 汇总逻辑（Phase 4 在主进程）

1. 等三个 subagent 都写完 json
2. 按字段合并：所有 `findings` 拼成 `summary`；`metrics` 合并进 `experts` 数组；`chart_suggestions` 合并供 HTML/PPT 使用
3. 构造统一的 `analysis.json`（schema 跟 `build_xlsx.py` 一致）
4. 三个专家名**只在 metrics 表里保留**（便于审阅），报告正文/HTML/PPT 不出现角色名

### 专家失败的兜底

- 任一 subagent 失败：`applicable = false` 或 timeout
- 主进程不阻塞其他专家；最终报告里不出现该视角的 section
- 终端打印 `[!] 结构分析师未能完成，跳过`

## 与动态选角的分工

| 场景 | 用哪套 |
|------|--------|
| AMD 视频演示 / 对演示稳定性要求高 | 本文档的**固定三专家**（`--experts all`）|
| 金融/医疗/行业数据，需专家陈述 | `workflows.md` 的**动态选角**（3-5 个） |
| 简单查数、单维度分析 | 不启用多专家，直接 `read_excel.py + LLM` |
