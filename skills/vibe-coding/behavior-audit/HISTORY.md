# 变更记录

## 2026-02-13: 初始创建

**变更**: 创建 behavior-audit Skill

**背景**: 基于对 AI 协作开发过程中经验沉淀的思考，发现需要一种结构化的方法来：
1. 全量记录开发过程（保真）
2. 多层过滤分析（提质）
3. 精准提炼 Skill（沉淀）

**核心设计**:
- 两级架构：记录阶段求全，沉淀阶段求精
- 四层过滤：时间范围 → 场景分类 → 信息提取 → 频率统计
- 人审核机制：AI 辅助分析，人最终确认

**参考**: 与 "浅聊 Skills、AGENTS.md 与经验萃取" 视频中的方法对比，
本 Skill 强调"全量记录 + 分层过滤"的工程化实践，更适合个人日常使用。
