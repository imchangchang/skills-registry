# Vibe Coding 方法论

## 核心理念（3条）

### 1. 约定优于配置

**原则**：用强制规则替代临时指令

- SKILL ≠ 参考资料（可看可不看）
- SKILL = 强制执行的工作约定（必须遵循）

### 2. 渐进式披露

**原则**：AI 按需加载，避免上下文爆炸

```
Level 1: Metadata（元数据）
    ↓ 触发条件匹配
Level 2: SKILL.md Body（核心流程）
    ↓ AI 判断需要
Level 3: References（详细资料）
```

### 3. 学习中演进

**原则**：被动式经验沉淀，能力持续成长

- 遇到问题才更新 SKILL
- 项目经验直接写入，无需清洗
- 能力层升级支撑更好的交付

---

## 双层结构

### 能力层（知识积累）

**两条输入路径**：

| 路径 | 来源 | 处理方式 | 时效 |
|------|------|----------|------|
| 主动学习 | 外部资料 | 需要数据清洗 | 较慢 |
| 实践沉淀 | 内生经验 | 直接写入 | 即时 |

### 交付层（项目迭代）

**流程**：
```
需求 → 方案讨论 → AI实现 → 质量门控 → 人验收
              ↑         ↓
           引用SKILL  反馈优化
```

---

## 三种约定层次

| 层次 | 定义 | 载体 | 特点 |
|------|------|------|------|
| 全局约定 | AI的"本性"原则 | vibe-coding/core | 跨领域通用 |
| 能力约定 | 领域内做事方法 | 各SKILL.md | 经验沉淀 |
| 项目约定 | 具体场景约束 | AGENTS.md | 临时性，可覆盖上层 |

---

## 五阶段实施流程

| 阶段 | 主导者 | 关键活动 |
|------|--------|----------|
| 1.需求与方案 | 人 | 明确需求，讨论方案，人做决策 |
| 2.技能准备 | 协作 | 识别所需Skill，加载相关约定 |
| 3.AI实现 | AI | 遵循约定生成代码，自测验证 |
| 4.质量门控 | AI | 自动化检查，确保基础质量 |
| 5.人验收 | 人 | 验证结果，决定迭代或沉淀 |

**验收后的选择**：
- 通过 → 结束
- 改需求 → 返回阶段1
- 优化方案 → 返回阶段3
- 沉淀经验 → 更新SKILL → 能力层提升

---

## 核心概念

### 中央技能仓库

- 存储所有 Skill 的权威来源
- 独立维护，与项目分离
- 项目通过符号链接只读引用

### 项目仓库

- 存储项目代码和特定上下文
- 生成 Skill 沉淀产物
- 不直接修改中央仓库

### 关系

```
项目仓库 --引用--> 中央技能仓库
    |                      |
    +-- 生成产物 ---------> 接收产物
```
